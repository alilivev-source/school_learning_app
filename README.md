# روضة النور 🌙 — تطبيق تعليم القراءة والكتابة للأطفال

تطبيق Flutter تعليمي ثنائي اللغة (عربي/إنجليزي) لتعليم الأطفال الحروف والكلمات
عبر بطاقات تفاعلية، ألعاب تعليمية (مطابقة، ذاكرة، بناء كلمات، تتبع حروف، اختبارات)،
قصص مصوّرة، ونظام تقدّم ومكافآت.

## تشغيل المشروع

```bash
flutter pub get
flutter run
```

> ملاحظة: نفّذ الأمر التالي مرة واحدة إن أردت أن يقوم Flutter بتوليد/تحديث أي
> ملفات منصّة إضافية (خاصة مجلد `ios/` غير الموجود حالياً، أو دعم ويندوز/ماك/ويب):
> ```bash
> flutter create .
> ```

## ملخص الإصلاحات والإضافات التي تمت

### 🐞 أخطاء برمجية تم إصلاحها
- **`effects_manager.dart`**: استخدام خاطئ لـ `WidgetsBinding.instance.context` (غير موجود أصلاً) داخل
  `getFireworksEffect()` و `getHeartsEffect()` — تم تحويلهما لاستقبال `BuildContext` كمعامل.
- **`effects_manager.dart`**: تمرير `List<Widget>` مباشرة إلى `Positioned.fill(child:...)` بدل `Widget` واحد
  — تم لفّها بـ `Stack`.
- **`effects_manager.dart`**: عمليات حسابية `int` تم تمريرها لمعاملات `double` (`fontSize`, `moveY end`)
  — تم تصحيحها بإضافة `.toDouble()`.
- **`letters_screen.dart`**: استخدام خاصية `opacity` غير الموجودة في `TextStyle` — استُبدلت بـ `Opacity` widget.
- **`memory_game.dart`**: استخدام `AppTheme` دون استيراد `app_theme.dart` (خطأ ترجمة) — تم إضافة الاستيراد.
- **جميع ملفات `assets/json/*.json`**: كانت تحتوي BOM (UTF-8 Byte Order Mark) في بداية الملف، ما يجعل
  `json.decode()` في Dart يفشل بصمت (الكود يبتلع الخطأ ضمن `try/catch` ويُرجع قوائم فارغة) — أي أن
  **كل الحروف/الكلمات/القصص/المستويات كانت ستظهر فارغة تماماً في التطبيق**. تم إزالة الـ BOM من الملفات الخمسة.
- **`pubspec.yaml`**: استيراد `flutter_localizations` مستخدم في `main.dart` لكنه غير مُعرّف كاعتمادية — تمت إضافته.
- **`pubspec.yaml`**: مسارات الأصول لم تكن تشمل `assets/svg/letters/arabic/` و `assets/svg/letters/english/`
  (فلاتر لا يُدرج المجلدات الفرعية تلقائياً) — تم إصلاح القائمة.
- **مشروع أندرويد غير مكتمل** (لن يُبنى إطلاقاً بالحالة السابقة):
  - `MainActivity.kt` غير موجود — تم إنشاؤه.
  - مجلد `res/` بالكامل (الأيقونات، الثيمات، شاشة البداية) غير موجود — تم إنشاؤه كاملاً.
  - `android/gradle.properties` كان فارغاً (بدون تفعيل AndroidX) — تمت إضافة الإعدادات القياسية.

### 🔌 فجوات وظيفية تم ربطها (كانت شاشات/ملفات معزولة لا تُستدعى من أي مكان)
- **شاشة الألعاب (`games_screen.dart`)** كانت تعرض رسالة "قيد التطوير" فقط ولا تُشغّل أي لعبة فعلياً —
  تم ربطها بتحميل بيانات الحروف/الكلمات وتشغيل الألعاب الخمس الفعلية.
- **متجر المكافآت (`reward_store_screen.dart`)** لم يكن له أي مسار تنقّل إليه من التطبيق — تمت إضافة
  مسار `/rewards` وزر وصول من الشاشة الرئيسية، وتفعيل خصم النجوم الفعلي عند الشراء
  (أضيفت `spendStars()` في `ProgressProvider`).
- **شاشة تفاصيل القصة (`story_detail_screen.dart`)** لم تكن مربوطة بقائمة القصص — تم ربطها.
- **حفظ نتائج الألعاب**: الألعاب الخمس لم تكن تحفظ نتيجتها في `ProgressProvider` رغم وجود
  `gameScores` في نموذج التقدّم — تمت إضافة الاستدعاءات اللازمة.
- **الوضع الليلي**: كان هناك مفتاح تخزين (`keyDarkMode`) مُعرّفاً ولا يُستخدم أبداً — تم بناء
  `SettingsProvider` + `AppTheme.darkTheme` وربطهما بالكامل مع مفتاح تفعيل في شاشة الإعدادات.
- **`flutter_local_notifications`** كانت اعتمادية مُعرّفة في `pubspec.yaml` ولا تُستخدم — تم بناء
  `NotificationService` لاستخدامها فعلياً (إشعار تذكير بالتدرّب).

### 🔤 الخطوط (Tajawal-Regular.ttf, Poppins-Regular.ttf)
لم يكن هناك اتصال بالإنترنت متاحاً في بيئة التنفيذ لتحميل الخطوط الأصلية من Google Fonts، لذا تم وضع
**خطوط بديلة مؤقتة** (FreeSerif لدعم العربية، DejaVu Sans للإنجليزية) بنفس أسماء الملفات المطلوبة حتى
لا يبقى أي ملف فارغ ويعمل التطبيق بصريّاً بشكل كامل فوراً. **يُنصح بشدة** باستبدال هذين الملفين بخطوط
Tajawal و Poppins الأصلية (متوفرة مجاناً على Google Fonts) قبل النشر، للحصول على المظهر المصمم فعلياً.
كذلك تمت إزالة اعتمادية `google_fonts` (لأنها تتطلب اتصال إنترنت لتحميل الخط أول مرة، غير مناسب لتطبيق
أطفال يُفترض أن يعمل بدون إنترنت) واستبدالها بالخطوط المحلية المرفقة في `assets/fonts/`.

## ⚠️ أشياء تحتاج انتباهكم قبل النشر

1. **مجلد `ios/` غير موجود إطلاقاً** في الحزمة الأصلية. لتشغيل التطبيق على آيفون، شغّلوا
   `flutter create .` في جذر المشروع (بوجود Flutter SDK مثبّت لديكم) لتوليده تلقائياً، ثم
   `flutter pub run flutter_launcher_icons` لتوليد أيقونة iOS أيضاً.
2. **الخطوط والأصوات الحالية بديلة مؤقتة** كما ذكرنا أعلاه — استبدالها بملفات احترافية موصى به بشدة.
3. **`applicationId`/`namespace`** في `android/app/build.gradle` لا يزال `com.example.rawdat_al_noor`
   — غيّروه إلى معرّف حزمتكم الفعلي قبل النشر على المتجر.
4. لم تتوفر لدينا بيئة تشغيل Flutter فعلية لتنفيذ `flutter analyze` أو بناء تجريبي؛ التدقيق تم يدوياً
   عبر قراءة كل ملفات الكود والتحقق من توازن الأقواس وصحة الاستيرادات ومطابقة الأصول. يُستحسن تشغيل:
   ```bash
   flutter analyze
   flutter pub get
   flutter run
   ```
   للتأكد النهائي بعد فتح المشروع لديكم.
