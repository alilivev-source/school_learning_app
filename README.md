# Mystic Dungeon 🗡️👹

لعبة محارب يخوض متاهات مليئة بالوحوش عبر 100 مرحلة متصاعدة الصعوبة، مبنية بـ Flutter + Flame.

## تشغيل المشروع

```bash
flutter pub get
flutter run
```

يدعم المشروع حاليًا **أندرويد** بشكل كامل. مجلد iOS غير مُهيّأ بملفات Xcode بعد
(راجع قسم "الملاحظات والقيود" بالأسفل).

## بنية المشروع

```

    weapons.dart                تعريف الأسلحة الأربعة وتبديلها
    game_status.dart            enum لحالة اللعبة (لعب/انتقال/إيقاف/خسارة/فوز)

      enemy.dart                 الوحوش والزعماء (ملاحقة أو تجوال عشوائي)
      projectile.dart            المقذوفات (سهم/سحر/قنبلة)
      collectible.dart           الجوائز (عملة/جوهرة/قلب/مفتاح/صندوق)
      maze_wall.dart             الجدران، الأرضية، وبوابة الخروج
      effect_sprite.dart         تأثيرات بصرية قصيرة العمر (انفجار/شرارة/غبار)
      floating_text.dart         نصوص عائمة (Combo، مكافأة هزيمة الزعيم)
  screens/
    main_menu_screen.dart       القائمة الرئيسية
    level_select_screen.dart    شبكة اختيار المرحلة (100 مرحلة)

  widgets/
    hud_overlay.dart            واجهة اللعب: صحة/صحة الزعيم/نقاط/عصا تحكم/أزرار

    status_overlays.dart        شاشات إيقاف مؤقت/خسارة/فوز نهائي

    save_service.dart           حفظ التقدم، تفضيلات الصوت/الموسيقى، بنك العملات، ترقيات الأسلحة

  sprites/{player,enemies,weapons,projectiles,effects,obstacles,items,ui}/

  sounds/

  level_generator_test.dart     اختبارات وحدة لمولّد المراحل
```

## آلية اللعب

- **الحركة**: عصا تحكم دائرية أسفل يسار الشاشة (كل الاتجاهات).
- **الهجوم**: زر دائري كبير أسفل يمين الشاشة، يطلق السلاح الحالي.
- **تبديل السلاح**: زر صغير يدور بين 4 أسلحة: سيف (قريب) ← قوس ← سحر ← قنبلة (انفجار بمساحة).
- **القفز**: نطة قصيرة مع حصانة مؤقتة من الضرر (مفيدة للتفادي).
- **الانزلاق (Dash)**: اندفاعة سريعة بالاتجاه الحالي مع حصانة قصيرة + أثر غبار.
- **الجوائز**: عملات وجواهر (تُضاف لبنك دائم يُستخدم بالمتجر)، قلوب (تعالج)، ومفتاح+صندوق كنز كل 5 مراحل.
- **زعيم كل 10 مراحل (Boss)**: وحش أكبر حجمًا وأقوى بكثير مع شريط صحة خاص أعلى الشاشة،
  يعطي مكافأة نقاط كبيرة عند هزيمته.
- **نظام Combo**: قتل أعداء متتاليين خلال نافذة قصيرة يرفع مكافأة النقاط ويظهر نص "Combo x2/x3..."
  عائم فوق العدو. يُصفَّر عند تلقي ضربة.
- **اهتزاز الشاشة + لمس اهتزازي (Haptics)**: عند الانفجارات، إصابة اللاعب، وهزيمة الزعيم،
  لإحساس قتالي أقوى على الجوال.
- **موسيقى خلفية + مؤثرات صوتية**: تُشغَّل تلقائيًا، ويمكن إيقاف كل منهما بشكل منفصل من قائمة الإيقاف المؤقت.
- **متجر ترقيات دائم**: بالقائمة الرئيسية، ترقّي ضرر/سرعة كل سلاح بالعملات المجموعة (حتى 5 مستويات لكل سلاح)،
  والترقية تبقى محفوظة حتى لو أعدت تشغيل التطبيق.
- **الانتقال بين المراحل**: عند الوصول للبوابة، يتحرك اللاعب تلقائيًا نحوها، ثم
  يتقلص ويدور داخل تأثير بوابة بصري، وتُحمّل المرحلة التالية تلقائيًا.
- **100 مرحلة**: تُولَّد إجرائيًا (خوارزمية Recursive Backtracker) بدل ملفات يدوية،
  مع تصاعد حجم المتاهة وعدد/قوة/سرعة الوحوش وضرر التلامس تدريجيًا. نفس رقم المرحلة
  يعطي دائمًا نفس التصميم (بذرة عشوائية ثابتة = levelNumber).

## ⚠️ ملاحظات وقيود مهمة (اقرأها قبل الاستخدام)

1. **الأصول الحالية Placeholder**: كل صور assets/sprites و assets/backgrounds
   مُولَّدة برمجيًا بأشكال هندسية بسيطة (flat-icon style) لأنني لا أملك أداة
   توليد رسوم فنية حقيقية في هذه البيئة. هي وظيفية بالكامل (تشتغل اللعبة 100%)
   لكنها ليست رسوم نهائية احترافية. لاستبدالها: احتفظ بنفس الأسماء والمسارات
   داخل assets/sprites/... وبس استبدل الملفات - لا حاجة لتعديل أي كود.
2. **الأصوات الحالية Placeholder أيضًا**: نغمات WAV بسيطة مولّدة رياضيًا (beep/thud)
   لتفعيل نظام الصوت بالكامل. استبدلها بنفس الأسماء داخل assets/sounds/.
3. **iOS غير مُهيّأ**: مجلد ios/ يحتاج مشروع Xcode كامل (Runner.xcodeproj) لم
   يتم إنشاؤه هنا لتجنّب توليد ملفات معقدة قد تفسد بسهولة بدون Xcode للتحقق منها.
   الحل: شغّل flutter create --platforms=ios . داخل هذا المشروع لإضافة دعم iOS
   تلقائيًا (لن يمس ملفات lib/ الحالية).
4. **لم يتم تشغيل flutter pub get / flutter analyze فعليًا** في بيئة التطوير
   هذه (لا يوجد Flutter SDK مثبت هنا)، فرغم المراجعة اليدوية الدقيقة لكل الأقواس
   والاستيرادات، يُنصح بتشغيل flutter analyze أول شيء بعد فتح المشروع للتأكد
   من عدم وجود أي ملاحظة بسيطة فاتتني.
5. **applicationId**: حاليًا com.example.mysticdungeon - غيّره قبل النشر على
   المتجر (في android/app/build.gradle).
6. **توقيع الإصدار (Release Signing)**: android/app/build.gradle يستخدم توقيع
   debug مؤقتًا حتى يعمل flutter run --release. أضف key.properties وتوقيعك
   الخاص قبل النشر الفعلي.
7. **أيقونة التطبيق**: مُصمَّمة (درع وسيف ذهبي على خلفية بوابة سحرية بنفسجية)
   وتدعم Adaptive Icon لأندرويد 8+ (`mipmap-anydpi-v26/ic_launcher.xml`) بالإضافة
   لنسخة PNG كلاسيكية تعمل على كل الإصدارات.

## أفكار لتوسيع اللعبة لاحقًا

- Sprite Sheets متحركة (Idle/Walk/Attack) بدل صورة ثابتة لكل حالة.
- تنويع بصري أكبر للزعماء (شكل مختلف لكل عشرة مراحل بدل نفس sprite النخبة).
- شاشة إنجازات/نجوم لكل مرحلة حسب الوقت أو الصحة المتبقية.
- لوحة صدارة (Leaderboard) عبر خدمة سحابية.
